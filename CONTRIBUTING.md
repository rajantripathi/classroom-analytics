# Contributing to classroom-analytics

Thank you for your interest in contributing to **classroom-analytics**.

## Ways to contribute

- Bug reports with a minimal reproducible example
- Feature requests with a clear use case
- Documentation improvements
- Code contributions with tests where practical
- Dataset or benchmark contributions, especially for low-resource languages

## How to submit a change

1. Fork the repository and create a feature branch.
2. Make a small, focused change with a clear commit message.
3. Test locally.
4. Update documentation if needed.
5. Open a pull request against the default branch.

## Code style

- Python 3.10+ unless the repository states otherwise
- Type hints are encouraged for public functions
- Prefer `ruff format` and `ruff check` where configured

## Code of Conduct

By participating you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).
